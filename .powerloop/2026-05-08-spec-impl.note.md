# Standing Principles

These apply to every cycle, dispatcher and SubAgents alike:

1. **REFERENCE alignment first**: `tmp/REFERENCE.md` is the validated design baseline for stage-1 implementation. Every Execute SubAgent prompt must instruct: "follow REFERENCE.md exactly; if a step seems impossible, report the specific blocker rather than improvising an alternate approach." When SubAgent reports a REFERENCE deviation, the dispatcher must challenge it — either it's a real blocker (record + escalate to user) or the SubAgent must redo per REFERENCE. Review-phase scanners must flag REFERENCE divergences as findings.
2. **Optimization only after first end-to-end working state**: Do not pre-optimize away from REFERENCE for binary size, perf, or ergonomics in stage-1. Measurement-driven changes happen after green E2E.
3. **Minimize custom implementation**: REFERENCE has been validated to reach the design goal **with minimal custom code** by leveraging existing libraries and APIs (msgpack gem, rmp crate, mruby C API, etc.). Re-implementing what REFERENCE delegates to a library is OFF-LIMITS in stage-1. When in doubt: less custom code beats more, and a tiny glue layer over a library beats a from-scratch reimplementation. SubAgent prompts must explicitly forbid hand-rolling what REFERENCE assigns to a library.
   - **Direction-error smell**: when you (dispatcher OR SubAgent) notice you're about to write N hundred lines of low-level X, STOP and re-check REFERENCE for the named library/API that handles X. Heavy hand-rolling is a smell, not an achievement.
4. **Priority override**: Items whose Notes column contains `priority:high` are picked by Execute phase BEFORE the lowest-pending-# rule. Within `priority:high` items, lowest # still wins. Use this for stage-1 alignment fixes that block all downstream work — never for ordinary feature items.
5. **Commit lock files alongside dependency changes**: `Cargo.lock` (root + every Cargo workspace), `Gemfile.lock`, and any other resolver-output lock file MUST be committed in the same commit as the dependency change. Lock files pin the resolved graph for reproducible builds. Do NOT gitignore them. Root `Cargo.lock` ships in the gem (per `bundle gem --ext=rust` baseline `git ls-files`-based selector); `Gemfile.lock` and `wasm/kobako-wasm/Cargo.lock` are committed but rejected from the gem package.
6. **Scaffold baseline = `bundle gem --ext=rust` first commit**: For gemspec, .gitignore, Rakefile, Gemfile shape, the reference is the project's first commit. Extend baseline mechanisms (e.g. extra reject paths in gemspec); do NOT replace them with custom allowlists or alternative structures.
7. **Do NOT cite `tmp/REFERENCE.md` in committed source code**. REFERENCE.md is a gitignored prototyping doc that never ships. Cite SPEC.md (committed) when needed, or omit. SubAgents may READ REFERENCE during implementation but their commits must not. The `.powerloop/*.note.md` file is exempt — process documentation, not shipped code.
8. **`test/` is gem-runtime-behavior only**: Minitest under `test/` targets the gem's user-observable behavior. Wrappers around external commands (`gem build`, `bundle exec rake <task>`, `cargo build`/`test`, `rubocop`, `grep`) do NOT belong in `test/` — the underlying command already exits non-zero on failure. Cross-language integration tests proving gem semantics neither side can verify alone (e.g. `test_codec_roundtrip_fuzz.rb`, `test_envelope_roundtrip.rb`) are the exception. For build/packaging/lint Execute items, the E2E requirement is "the underlying command exits 0", not "wrap it in Minitest".

Operational hygiene (NOT principles, just dispatcher reminders): keep `.note.md` log rows chronological by appending at the bottom; verify the three headings (`# Standing Principles`, `# Progress`, `# Log`) survive any structural edit; cite REFERENCE line numbers in audit-driven items so the source-of-truth reasoning is traceable. These are too operational to formalize as Standing Principles — just be careful.
